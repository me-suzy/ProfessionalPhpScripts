System Requirements  

  • UNIX or Windows 95/98/NT/2000 Web Server
  • Perl 5.0 or higher
  • Perl Modules Required:
    • CGI.pm
    • LWP.pm
    • Socket.pm

SimpleSubmit will not run, displaying a 500 Server Error if you have an outdated version of Perl installed. If you have a current version of Perl, but are missing some modules, SimpleSubmit will tell you which modules are missing. In general, your system admin will need to install Perl and the required modules. If you administrate your own server, you can download Perl and all required modules from http://www.cpan.org/ or http://www.activestate.com/

Technical Support 

Technical support for VerySimple scripts is provided through the VerySimple support forum at http://www.verysimple.com/public/ If you have questions or problems, you may find it already answered there. If it is not answered, you can post a question and receive e-mail when an answer is posted. Because VerySimple is mostly the spare-time work of one person, there is no company to call or e-mail for support. VerySimple software depends upon the support of it's users. Since this software is offered for free, please consider visiting the support forum and contributing your knowledge.

About 500 Server Errors: Before you post a message or inquire about a 500 server error, you will need more information. A 500 server error is not enough information to troubleshoot your problem. It basically means that the web server has no idea what went wrong, so it didn't display the page. To see what the problem is, you need to see what the real error message is. There are two ways to find the real error message: check the server error logs, or run the script from the command line.  The location of your server error logs depends on your server configuration.  (See "Running SimpleSubmit from the UNIX or DOS command line" below for more information)

Installation

Download submit.zip to your local computer or directly to your web server and extract the files using your preferred zip unarchiving software.

  1. Upload or copy all files to your web server. (FTP users: Be sure to use ASCII mode) Save the files in a directory that allows web visitors to access and execute CGI scripts, such as /cgi-bin/. (Contact your system admin or refer to your host site support pages if you don't know)
  2. If necessary, change the file permissions allowing web visitors to execute submit.cgi and submit.setup.cgi. (UNIX systems generally require you to set permissions. Windows systems generally do not.)
  3. Open submit.setup.cgi using your preferred web browser. (for example, http://www.yoursite.com/cgi-bin/submit.setup.cgi

Setup Options

When you first run the SimpleSubmit setup application, you will be prompted to run standard setup, advanced setup or update the .tab file.  It is recommended that you first use standard setup to get the script up and running.  Once you have the script working properly, you may want to go back and customize it using advanced setup. Standard setup requires only one configuration variable and installs SimpleSubmit with the options that are most common for personal use.  Standard setup installs SimpleSubmit using the "Ban All" security mode, no "thank-you message", no usage logging and maximum of 10 URLs at one time.

If you use SimpleSubmit as a service to other webmasters and/or promote it as your own service, you will need to run advanced setup. You will need to provide more information to get started, however you can run the setup program as many times as you like and your previous settings will be retained. If you should ever wish to restore the default settings, simply run the standard setup.

Advanced setup lets you choose between the two security modes: "Ban All Domains by Default," or "Allow All Domains by Default." To prevent abuse of you server, it is recommended that you choose "Ban All" unless you understand what you are doing. "Ban All" means that only domains which you specify may use SimpleSubmit on your server. This prevents abuse because you have total control over which domains are allowed to submit. "Allow All" mode does the exact opposite. All domains are allowed by default unless you specifically ban them.  It is necessary to use "Allow All" if you want to provide SimpleSubmit as a service to other webmasters, since you don't know the domain names of the people who will be using your service.  However, "Allow All" opens your site to possible abuse. If you choose this security mode, it is recommended that you also enable logging and monitor the log file regularly.

Setup Variables

  • Authorized Domains: In "ban all" security mode, this provides the list of all domains that can use SimpleSubmit.
  • .cfg File Location: Advanced setup allows you to rename or place the .cfg file in an alternate directory. The path is relative from the directory where submit.cgi is located.
  • Banned Domains: In "allow all" security mode, this is a list of all domains that are banned from using SimpleSubmit.
  • Sendmail Path/SMTP Server: In order to send the e-mail thank-you message, a path to sendmail or SMTP server is needed. You only need to provide one. Setup will attempt to determine the best option based on your operating system and domain name. If do not want to send a thank-you message or your system doesn't support it, then leave both of these fields blank.
  • E-mail Template File: This is the file that is used as a template for the thank-you message. You can customize this file with your own personalized thank-you message.
  • E-mail Message is Sent From: This is the senders e-mail address for the thank-you message.
  • E-mail Subject: The subject of the thank-you message.
  • E-mail Message Format: You can send the thank-you message in plain text or HTML format.
  • Log File: SimpleSubmit will track all usage to a file in tab-delimited format. If you do not wish to log usage, leave this field blank. Note that you should monitor this file if you have logging enabled. The file can grow large if you have a busy site. To open the logfile, you can use any database or spreadsheet program that imports tab-delimited files (such as MS Excel, MS Access, etc.).
  • Maximum URLs: To prevent abuse of your server, you can specify the maximum number of URLs that can be submitted at one time.
  • Page Header/Footer Template File: The design of SimpleSubmit can be completely customized by editing the page header and page footer files. If you open these files with a text editor, you will see that they are simply standard HTML files. The header file is included at the top of the SimpleSubmit page and the footer file is included at the bottom.
  • Border Color: The background color of the horizontal separator bars. You can customize these to match your custom design.
  • Border Font Color: The font color of the horizontal separator bars.
  • .tab File Location: The location of the .tab database file.

Updating The .tab File

The .tab file is a database containing information about the search engines. Since each search engine is owned by a difference company, their methods for submitting sites will vary. Most search engines change this method from time to time. Also, new search engines are added periodically to the database. By maintaining all of this information in a separate database file, SimpleSumbit can be updated easily without reinstalling the script. To see what .tab version you have, you can enter "info" (without the quotes) in the URL field of simplesubmit.

To update your .tab file, you can simply run the setup program and select the third option "Update .tab File Only" The setup program will download the latest .tab file from verysimple.com and overwrite your existing .tab file. WARNING: If you have customized your .tab file, please be sure to make a backup copy before updating!

You can also add your own search engines to the .tab file. See the Customizing section of this page for more inforamation.

Usage

Running SimpleSubmit From The Browser

Using the SimpleSubmit browser interface is easy. Open submit.cgi in your preferred web browser (for example http://www.yoursite.com/cgi-bin/submit.cgi). Enter one or more URL's in the URL field (one per line). Enter your e-mail address in the e-mail field.

If you wish to use the default search engines, then you can click submit at this point. If you want to submit only to specific search engines or one that is not a default engine, then you can click on only those engines you wish to submit.

If you don't want a thank-you message, then you can unselect that checkbox.

Running SimpleSubmit From the UNIX or DOS command Line:

SimpleSubmit can be run from the command line as well as through a web browser. To do this, you must have Perl installed. (Windows users can download Perl from ActiveState) All of the features of SimpleSubmit are available through the command line, however it is easier to use the browser interface. It may be preferable to use the command line interface, however, especially if you wish to schedule SimpleSubmit to run automatically.

Usage:

perl submit.cgi url=http://site.com/ email=me@site.com [confirm=yes] [engines=a,b,c]

[brakets] indicate optional fields. Specify multiple sites and multiple engines by separating them with a comma. If you do not specify command line parameters when you run the script then SimpleSubmit will promt you. Enter 1 parameter per line and press <CTRL> d (for UNIX) or <CTRL> z (for DOS) when finished.

Scheduling SimpleSubmit to Run Automatically:

One possible use of SimpleSubmit is to automatically schedule it to run and submit your site on a regular basis. (A word of caution: don't schedule your submissions too frequently or the search engines may permanently block your site. Once per month or less is gererally sufficient)

To schedule SimpleSubmit to run on UNIX, you can use the cron utility. Usage rules vary from host to host, so check with your admin and/or support pages. To schedule using Windows/DOS, you can use the built-in task scheduler. Refer to the Microsoft help file for more inforamation.

When you schedule SimpleSubmit to run automatically, it is usually helpful to have the normal output go to a file instead of to the screen - since you won't be there to see it. This can help you troubleshoot and/or see that the last submission went as planned. The email confirmation is also helpful, as it will email you the results.

To send the output to a file, you simply append "> filename" to the command line. For example:

perl submit.cgi url=http://site.com email=me@site.com > /usr/site/submitresult.txt

Customizing SimpleSubmit

Colors and Layout:

You can completely customize the look of SimpleSubmit to match your own brand or site design. Two configuration files contain the HTML code that is used by SimpleSubmit. By default, these are submit.hed and submit.fot and they are located in the same directory as submit.cgi. (You can change the name and location of these files using advanced setup).

submit.hed contains all the HTML code at the top of the page and submit.fot contains the code at the bottom. submit.cgi automatically generates all of the other code. Although you cannot customize the automatically generated code, you can specify the color of the borders and fonts in advanced setup.

Edit submit.hed and submit.fot to match the design of your existing site.

NOTE: although it is possible to create a design using tables and put the SimpleSubmit automatically generated text within this table, it is not recommended. The reason for this is that SimpleSubmit will display the search engine results in real-time. If you are submitting to a large number of engines, then it is helpful to see the progress. Putting the results in a table usually will cause the browser to wait until the entire page loads before displaying anything. If you do this, it will appear that SimpleSubmit is not loading, when in fact it is just taking several moments to contact all of the search engines. For this reason, it is recommended that you do not enclose the results in a table. If you need to do this to accomodate your site design, it is recommeded that you use frames instead.

If you need to customize the main SimpleSubmit page beyond what is allowed using submit.hed and submit.fot, you can do that as well. Simply open submit.cgi in your browser and save the HTML source. Customize it as much as you like. You can then use this page as the form interface to SimpleSubmit instead of the autmatically generated page. Note, that if you update your .tab file, you may need to edit your custom design to add or remove any search engines from the select box.

Thank-You Message:

In advanced setup you can specify that a thank-you message is sent when people use SimpleSubmit. (Sendmail or SMTP is required to use this feature) The content of this message is generated from a template file, submit.tha. You can edit this file to your liking.

You can specify that that thank-you message is sent in Plain Text or HTML format in advanced setup. All e-mail clients can read plain-text. HTML format allows you to create a formatted message with layout and graphics, however not all e-mail clients support HTML e-mail. If you specify HTML format, then you will want to design submit.tha as an HTML document.

Add Your Own Search Engines:

SimpleSubmit can be enhanced by adding your own search engines.  SimpleSubmit come pre-installed with most of the major search engines, but you can add value by customizing it with your own selection of search engines.  Imagine if you have a sports-related site.  You can customize SimpleSubmit to submit to all the sports-related search engines on the web.  There are hundreds of search engines and link pages for all topics.  SimpleSubmit will work with the vast majority of them.  You can offer highly targeted promotion to your visitors, or simply use SimpleSubmit to make your own promotional efforts easier. 

All the information about each search engine is contained in the file submit.tab.  This is simply a tab-delimited database file containing information about each engine. The file can be opened with any text, spreadsheet or database application tat supports tab-delimited files (such as MS Excel or MS Access).  You can even open it using notepad, however it can be tricky to read.

The .tab file has several fields that are used to tell SimpleSubmit how each particular engine works. To add a new engine, you simply add a new line to the end of the file and fill in each field with the appropriate information.

To add a new engine, you will have to visit the site that you want to add and do a bit of detective work. Find the page that the engine provides for submitting sites. This page will contain a form with several fields and an "action" URL that the form uses to process the form. View the source of this page, submit your site and examine how the engine works.

Adding new engines might seem complicated at first. By looking at the .tab file, visiting a few of the default search engines and submitting a page directly from their site, you should be able to see how it works. The good news is that most search engines are very similar. Once you add one new engine, it will be easy to add more.

TabFile Fields:

  • Code: This is a unique code you assign to each engine. It is best to use only characters or numbers (no spaces).
  • Category: Category for each engine. Currently this field is not used, but future support for multiple categories is planned.
  • Description: A brief description of the seach engine.
  • HomeUrl: The URL to the search engines home page. Currently this field is not used, but future support is planned. This is also helpful information to have if the submission method changes.
  • LogoUrl: A URL to a graphic logo for the site. Currently this field is not used, but future support is planned.
  • SubmitUrl: The URL to the submission processing page. Note, this is not necessarily the page where their submission form is, but the actual program or script that is used to process submissions.
  • Method: Method is either GET or POST. This refers to the submission method requred by the search engine.
  • Varmap: Varmap tells SimpleSubmit what the variables names are for this search engine. This tells SimpleSubmit how to send the URL, Email and any other fields to the engine. See below for more details about Varmap.
  • StaticVars: StaticVars are any variables that the search engine requires to process the submission that are not provided by the webmaster. As an example example, a certain search engine may require "action=submit" for every submission so that the form processor know what to do with the data. You can add as many static variables as necessary.
  • Success: So that SimpleSubmit knows whether or not the submission succeeded, it parses the results from the search engine and looks for a "Success" message. This is simply any bit of text that appears on the search engines results pages when the submission is successful. For example, a certain engine may display "Thank You" when your submission was accepted. Be sure that the text you specify does not appear if the submission failed, or else SimpleSubmit will always think that the submissions to this engine were successful.
  • Default: 1 = yes, 0 = no. This specifies whether this search engine is a default when submitting your sites.

More About theVarmap Field:

The VarMap field may seem confusing at first, but if you look at the tab file and examine the search engines, you'll see that it is actually not that complicated.

SimpleSubmit essentially allows you to forward the information in one HTML form to multiple search engines. However, each of those engines has a different system for accepting input. For example, Altavista expects the URL field to be called "q", while Excite expects a field called "url." SimpleSubmit needs to know how to send your information to each unique engine. To do that, the Varmap field is used to map the fieldname on the SimpleSubmit HTML form to the fieldname used by the search engine.

On the SimpleSubmit HTML form, the URL field is called "url" and the email field is called "email." To add a new engine, we simply need to know how these two fields map to the new search engine. For example, say we add an engine that expects a field "p" to contain the URL and a field "e" to contain the email address. In this case, the Varmapforthisenginewouldbe"url=p,email= e"  Say we add an another engine that expects a field "site" to contain the URL and a field "contact" to contain the email address. In this case, theVarmapforthisenginewouldbe"url=site,email=contact" 

If the engine doesn't need the email address, then you can simply leave it out of the Varmap field.

Adding Sites That Require Additional Fields:

If you want to add an engine that requires more fields than just URL and Email, then you can simply add those to the Varmap field as well. However, you will have to create a custom SimpleSubmit form to use as well. For example, say you want to add an engine thatexpectsURL,EmailandSiteTitle.TheVarmapmightlooksomethinglikethis"url=p,email=e,title=t." After you add the engine to the .tab file, you'll have to create a form that has a field for the title in addition to the regular SimpleSubmit fields.
