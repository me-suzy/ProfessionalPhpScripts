a) Overview

StatIt is a script written in PHP. It is a web page logger that keeps track
of visitor count and other information such as system, browser, referal
domains, screen resolution, color etc. A web-based statistics page is available.
MySQL is not needed but the script is fast and the logfiles need not too many diskspace!
The scripts are PHP3 conform and should be work on almost every PHP3 or higher server!
Minimum PHP3.0.8 required!

This will be shown:

- visitors totay
- prediction of daily visitors
- visitors yesterday
- monthly average of visitors
- visitors for the actual month
- total (and unique) visitors
- the best month, workday, hour, screen resolution, screen color, browser, system, referer,
   searchengine query, top level domain and site
- the best day with count
- the last visitors with IP-address, hostname, referer, system, browser, language
- the visitors of the actual month
- the visitors of the day
- the visitors of the work day
- the visitors of the year (only one year will be shown)
- the browser of the visitors
- the system of the visitors
- the mostly frequent referer
- the screen resolution
- the colors of the screen
- the searchengine queries
- the probability of visitors of a day
- the size of the log files
- the origin of the visitors (top level domain and continent)

others:

- authentification for the statistic is possible
- to count single web pages is possible
- visible counter available (graphical or text)
- admin tool available
- reload blocker available
- statistic in this languages available:
german, english, dutch, portuguese, spanish, french, indonesian, italian, polish, hungarian

Any translations are welcome!

c) Installation

a)   Install script (still beta)
b)   Install manually
  1.   You have a PHP server.
  1.1. You work with frames.
  1.2. You work without frames.
  1.3. You want to track only the start page (e.g. index.php3).
  1.4. You want to track all PHP files on your server.
  1.5. You want to have a visible counter
  2.   You have your files on a server without PHP support but you have another server with PHP support.
  2.1. You work with frames.
  2.2. You work without frames.

a) Install script (still beta)

a) Upload all files (include directories normal, simple and language)!
c) chmod the directories to 777!
d) Run install.php3 (rename install.php3 to install.php if you have a PHP4 server and the script won't run
   but not rename any files without the install.php3 before running the install script!!)
e) Follow the install instructions!
f) If you have any problems with the install script try the manual install!

b) Install manually

1. You have a PHP server.

a) Edit _config.inc in dir normal, if needed (other languages in dir /languages)!
b) Edit statit.php3, statitp.php3, showcount.php3 in dir normal (set absolut path in require function)!
If you don't know your absolut path run abspath.php3!
c) Edit _short.inc in dir normal (set absolut path)!
d) Edit _lopa.inc.php3 in dir normal (set login and password)
You work with frames or you work without frames.

1.1. You work with frames.

a) Add

<script language="javascript">
<!--
document.write("<img width=1 height=1 border=0 src=path/statitjs.php3?1,"+screen.width+","+screen.colorDepth+">");
//-->
</script>

between the body tags of a part of your frameset (e.g. menue.html or menue.php3)!
b) Add

<?php include "path/statit.php3" ?>

to the calling page (e.g. index.php3)! If you want to count unique visitors the include must be in the first line before any HTML code!
The calling page must be a php page and must be on the same server!
A http:// in path doesn't work!!!
You want to track only the start page or you want to track all PHP files.

1.2. You work without frames.

a) Add

<script language="javascript">
<!--
document.write("<img width=1 height=1 border=0 src=path/statitjs.php3?1,"+screen.width+","+screen.colorDepth+">");
//-->
</script>

between the body tags in the calling page (e.g index.php3) and

<?php include "path/statit.php3" ?>

to the calling page (e.g index.php3)! If you want to count unique visitors the include must be in the first line before any HTML code!
The calling page must be a php page and must be on the same server!
A http:// in path doesn't work!!!
You want to track only the start page or you want to track all PHP files.

1.3. You want to track only the start page (e.g. index.php3).

a) Edit _short.inc in dir normal: Set $st_statp = "off";
Set the absolut path to variables! If you don't know the absolut path run abspath.php3!!
b) Create a directory (e.g. stat)
c) chmod directory to 777
d) Upload the files from dir normal to directory (image files in BINARY mode, other files in ASCII mode)
e) If you update from older version run update.php3 to change file format in stat.log!
f) Run statistik.php3 to show statistic of your website!

1.4. You want to track all PHP files on your server.

a) Edit _short.inc: Set $st_statp = "on";
Set the absolut path to variables! If you don't know the absolut path run abspath.php3!!
b) Add

<?php include "path/statitp.php3" ?>

to all PHP pages you want to track also the file where you added the <?php include "path/statit.php3" ?>
A http:// in path doesn't work!!!
c) Create a directory (e.g. stat)
b) chmod directory to 777
c) Upload the files from dir normal to directory (image files in BINARY mode, other files in ASCII mode)
d) If you update from older version run update.php3 to change file format in stat.log!
e) Run statistik.php3 to show statistic of your website!

1.5. You want to have a visible counter.

a) Edit _short.inc:
You can change the counter type (text or graphic), the number of digits and if you choose text, you can change the font, colors and size!
If you choose graphical counter you have to set the picture path and the type (.gif, .jpg etc.)!
b) Add

<?php include "path/showcount.php3" ?>

to the PHP page you want the visible counter!
A http:// in path doesn't work!!!
Place the code after the <?php include "path/statit.php3" ?> if you place it in the same page!

a) Edit _config.inc in dir simple, if needed (other languages in dir /languages)!

d) Administration

With the script admin.php3 you can:

- reset the log files domain.log, country.log, browser.log and system.log
- make backups of log files
- reset screen resolution, screen colors, work days and day statistic in stat.log
- change login and passwort
- change HTTP-authentification (on or off)
- change max size of the bars ( height and width)
- change IP-address showing (on or off)

ATTENTION: Change the login and password in the _lopa.inc.php3 or everyone who know StatIt can reset your statistic!!
If the auth function doesn't work on your server you can't use the admin script!

e) Known problems

a) If you have a PHP4 Server and the script doesn't work, try to rename all files from .php3 to .php!
   But if you install the script via install.php3 only rename the install.php3!
b) Notice: It's impossible to recognize all browsers or systems correct!
c) In some cases referers will not linked in the "last 20 visitors table" although they are attainable!
d) If you are behind a firewall the script have some problems!
e) If you got an 500 error from the server when you run admin.php3 your server doesn't support HTTP-auth!
